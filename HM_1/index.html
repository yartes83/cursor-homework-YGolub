Шпаргалка по використанню Git

Перелік основних команд Git:

  • git config --global user.name blabla - налаштування імені користувача.
  • git config --global user.email blabal@bla - налаштування e-mail користувача.
  • git init - створює локальний репозиторій Git.
  • git status - стан файлів.
  • git add - додавання файлів для наступного коміта.
  • git commit - знімок поточного стану змін, доданих до розділу проіндексованих файлів.
  • git log - перераховує комміти, зроблені в репозиторії.
  • git checkout - дозволяє переміщатися між гілками, створеними командою git branch.
  • git branch - дозволяє створювати, переглядати, перейменовувати та видаляти гілки.
  • git push - вивантаження вмісту локального репозиторію у віддалений репозиторій.
  • git pull - видалення та завантаження вмісту віддаленого репозиторію та негайного оновлення локального репозиторію цим вмістом.
  • git merge - використовується для злиття однієї або декількох гілок у поточну.
  • git clone - вибір існуючого репозиторію з подальшим створенням його клона чи копії.
  • git revert - скасовує внесені зміни та додає новий коміт з отриманим вмістом.
  • git reset - скасування змін у розділі проіндексованих файлів.
  • git rebase - спосіб перенести зміни з однієї гілки до іншої. Rebase стискає всі зміни в один патч. Потім він інтегрує патч у цільову гілку.
  • git cherry-pick - бере зміни, що вносяться одним коммітом, і намагається повторно застосувати їх у вигляді нового комміту в поточній гілці.
  • git stash - призначена для того, щоб помістити поточні зміни, які ви здійснили у файлах, в окреме сховище.

.gitignore - файли відстежуються у спеціальному файлі .gitignore, який реєструється у кореневому каталозі репозиторію.
У Git немає спеціальної команди для вказівки файлів, що ігноруються: замість цього необхідно вручну відредагувати файл .gitignore, щоб вказати в ньому нові файли, які повинні бути проігноровані
